# DISPATCH MODE — интерпретация сигналов канала «📈» → предложения сделок

**Когда активируется:** пользователь присылает (предварительно отвалидированное)
сообщение из крипто-канала. Моя задача — превратить его в **структурный сигнал**
и, если он торгуемый, отправить **предложение Neo-сделки** в Telegram-бот
пользователя. **Ничего не исполняется автоматически — решает пользователь.**

---

## 0. Золотые правила (не нарушать)

1. **Никогда не предлагать сделку на КОММЕНТАРИИ/прогнозе** — только на явном входе.
2. **Сомневаешься в инструменте/направлении → спроси пользователя**, не угадывай.
3. **Всегда 1 лот по умолчанию**, маржинальный гард в `send_proposal.py` блокирует
   слишком крупные (ГО × 2 > свободной маржи).
4. **Логируй каждый сигнал** (даже пропущенный) — для честной форвард-статистики.
5. Я предлагаю — пользователь подтверждает в приложении Т-Инвестиции сам.

---

## 1. Стиль автора канала (важно понимать!)

Дискреционный свинг-трейдер. Пишет криптово, фрагментами, часто задним числом
хвалит свои прогнозы. **Доминирующий стиль — шорты в падение.** Уроки на ошибках:

### Направление
- **SHORT:** `Short`, `шорт`, `short'чу`, `по'short`, `продаю`, `в шорт`, `Short'ил`
- **LONG:** `Long`, `лонг`, `лонганул`, `в лонг`, `лимитка снизу`, `Long'анул`

### ACTIONABLE (это ВХОД — можно предлагать) — есть глагол открытия:
`открыл`, `зашёл`, `шортю`, `шортчу`, `лонганул`, `поставил лимитку`,
`пробуйте Short/Long`, `в шорт/лонг с …`

### НЕ actionable (КОММЕНТАРИЙ/хайндсайт — НЕ предлагать):
`Готово ✅`, `Реализовано`, `Цель достигнута ✅`, `так и получилось`,
`ждём…`, `weekly review`, `не перепутайте`, `обратите внимание`,
`продублирую`, `актуальный нарратив`, чистые прогнозы цен без «открыл».

> ⚠️ **КЛЮЧЕВОЙ УРОК:** `«Теперь ждём, чтобы BTC дал отскок выше ✅»` — это
> **ОЖИДАНИЕ/комментарий, НЕ лонг-вход.** Он ждёт отскок, чтобы ШОРТить в него.
> Я однажды прочитал это как лонг → −14% в бэктесте. НЕ повторять.

### Картинки (PnL-скрины BingX/Bybit)
Показывают его **реальную позицию**: инструмент + направление + плечо.
Используй их как подтверждение: `«До $1540 открыл Short»` + ETH-скрин (2000→1639)
⇒ **ETH short**. Реф-код на скрине добавляется биржей автоматически (не признак скама).

### Маппинг «голый уровень → инструмент» (по диапазону цен)
| Уровень в тексте | Инструмент |
|------------------|------------|
| ~$60 000–$95 000 | **BTC** |
| ~$1 500–$2 600   | **ETH** |
| ~$60–$95         | **SOL** |
Пример: `«под 75k»` ⇒ BTC; `«$1540»` ⇒ ETH; `«$81.5-82»` ⇒ SOL.

### Многошаговые посты
`«сброс → вынос вверх во вторник → снижение во второй половине недели»` —
извлекай только **ближайшую торгуемую ногу** или эскалируй пользователю.
Тезисы («в июне падение») = **directional BIAS**, без входа пока он сам не открыл.

## 2. Схема сигнала (структурный вывод)

```
instrument : BTC | ETH | SOL | XRP | TRX | TSLA | NBIS | CVNA | APP | HOOD
neo_ticker : <X>USDperpA / <X>perpA   (см. таблицу ниже)
direction  : +1 (long) | -1 (short) | 0 (нет)
actionable : true/false   — это свежий вход?
confidence : high | low   — low ⇒ моя интерпретация/вопрос пользователю
levels     : [числа из текста]
reason     : короткое обоснование
```

## 3. Рабочий процесс (по шагам)

**Шаг 1 — первичный триаж (эвристика):**
```
python -u -m futbot.signals.interpreter      # smoke-test
```
или мысленно применяю `interpret(text)` (futbot/signals/interpreter.py):
HIGH = чёткий инструмент+направление+глагол входа; LOW = нужна моя интерпретация.

**Шаг 2 — разрешаю криптовые (LOW):** применяю правила §1 (картинки, уровни,
контекст). Если осталась двусмысленность по инструменту/направлению — **спрашиваю
пользователя одним вопросом**, не угадываю.

**Шаг 3 — если ACTIONABLE → проверяю рынок через t-invest MCP** (§4) и **отправляю
предложение:**
```
python -u -m futbot.scripts.send_proposal \
    --instrument ETH --dir short --lots 1 --stop-pct 6 \
    --note "759: ETH short, target $1540" --msg-id 759
```
Скрипт сам: тянет цену Neo + курс USD/RUB + свободную маржу, считает ГО, проверяет
буфер (ГО×2 ≤ своб.маржа), считает стоп, **шлёт в Telegram-бот**, пишет в
`data/signals.db`. Если маржи мало → пометит `❌ МАРЖА МАЛА` (всё равно отправит,
но видно что нельзя).

**Шаг 4 — если НЕ actionable** → НЕ шлю предложение; коротко говорю пользователю
почему (комментарий/тезис/двусмысленно) и логирую как `skipped`.

## 4. Интеграция с t-invest MCP (для быстрой проверки)

- **Цена Neo:** `mcp__t-invest__get_last_prices` с FIGI (таблица ниже) — или
  `mcp__t-invest__find_instrument` по тикеру.
- **Маржа/портфель:** `mcp__t-invest__get_portfolio_summary` (ликвидный портфель)
  и `mcp__t-invest__get_positions` (открытые позиции — нет ли уже этой).
- **Сверка после входа:** `mcp__t-invest__get_operations` (фактические филлы в USD,
  расчёт в РУБ по курсу закрытия).
- `send_proposal.py` сам делает расчёт ГО/маржи/стопа — MCP нужен мне для
  быстрой ручной проверки/контекста.

### Neo-инструменты (тикер → FIGI → база)
| Симв | Тикер | FIGI | Плечо≈ |
|------|-------|------|--------|
| BTC  | BTCUSDperpA | BTCUSDPERP00 | ~6.5× |
| ETH  | ETHUSDperpA | ETHUSDPERP00 | ~2.7× |
| SOL  | SOLUSDperpA | SOLUSDPERP00 | ~2.6× |
| XRP  | XRPUSDperpA | XRPUSDPERP00 | ~3.3× |
| TRX  | TRXUSDperpA | TRXUSDPERP00 | ~4.5× |
| TSLA | TSLAperpA   | TSLAPERP0000 | ~6.5× |
| AMD  | AMDperpA    | AMDPERP00000 | ~2.8× |
(плечо = 1/ставка_риска, ставка динамическая — точную берёт send_proposal из API)

## 5. Памятка по риску неоактивов

- Цена в USD, **P&L в РУБ по курсу закрытия** (`(выход−вход)×лоты×курс`).
- **Комиссия за каждую ночь** (00:00 МСК), а не за часы. Открыл+закрыл в день → 0.
- **Нет экспирации** (бессрочные).
- **Маржин-колл** = ликвидный портфель ниже минимальной маржи → принудительное
  закрытие. Гард (буфер 2×) + стоп 6% не дают приблизиться.
- ETH-неоактив крупный (ГО ~43k на 1 лот) — на счёте ~50k часто **не влезает**:
  гард заблокирует. SOL/XRP/TRX/дешёвые крипты — влезают.
