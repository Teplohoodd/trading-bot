\section{Overview}

The system is a single-process Python 3.11 application running an
\texttt{asyncio} event loop. Inside the loop, four concurrent coroutines
own distinct concerns:

\begin{enumerate}
  \item \textbf{Autonomous loop} — periodic screen $\to$ signal $\to$
        risk gate $\to$ execute pipeline.
  \item \textbf{Position monitor} — software stop-loss / take-profit
        fallback, trailing stop, time exits, signal-reversal exits, and
        partial take-profit.
  \item \textbf{Portfolio sync} — reconciliation with the broker;
        catches broker-side stop fills and infers \texttt{exit\_reason}.
  \item \textbf{Retrain scheduler} — daily LightGBM training with a
        rollback gate.
\end{enumerate}

A separate Telegram \texttt{Application} runs in the same loop with a
heartbeat watchdog that force-restarts polling on extended network
outages.

\section{ML pipeline}

\subsection{Triple-barrier labels (López de Prado, ch.\,3)}

Each candle in the training set produces a label by simulating a
forward-looking trade with three barriers:

\[
\texttt{PT}_t = c_t + \alpha \cdot \mathrm{ATR}_t,\qquad
\texttt{SL}_t = c_t - \alpha \cdot \mathrm{ATR}_t,\qquad
\texttt{T}_t = t + h
\]

where $c_t$ is the close, $\alpha$ is the symmetric multiplier (default
$\alpha = 1.0$ to keep classes balanced), and $h$ is
\texttt{TB\_MAX\_HOLD} (20 bars for shares, 10 for futures). The label
is the index of the first barrier touched, mapped to
\{sell, hold, buy\}.

The horizon $h$ is selected by IC (Spearman rank correlation) grid
search rather than F1 — at $h{=}1$ the hold class dominates ($93\%$),
inflating F1 to $0.91$ but with $\mathrm{IC}=0.07$.

\subsection{Per-asset-class isotonic calibration}

Raw LightGBM probabilities are biased on imbalanced classes. After
training, an out-of-fold prediction set is used to fit
\texttt{IsotonicRegression} \emph{separately for $P(\text{buy})$ and
$P(\text{sell})$ on each asset class}. Asset classes are computed from
\texttt{instrument\_kind} and ticker prefix (BR, GZ, GD, SR, RI, MM,
shares, currency, other).

Inference path:

\begin{lstlisting}[language=Python]
raw_probs = lgbm.predict_proba(X)             # shape (n, 3)
asset_class = get_asset_class(ticker, kind)
p_buy_cal  = calibrators[asset_class]["buy"](raw_probs[:, 2])
p_sell_cal = calibrators[asset_class]["sell"](raw_probs[:, 0])
# Re-normalise so probabilities still sum to 1.0
\end{lstlisting}

\subsection{Meta-labelling}

A secondary binary classifier learns to predict whether the primary's
direction will be correct, conditioned on the primary's confidence and
the same feature set:

\[
\text{final\_conf} = \sqrt{\text{primary\_conf} \cdot \text{meta\_conf}}
\]

Trades are gated on
$\text{meta\_conf} \geq \texttt{META\_LABEL\_THRESHOLD}$ \emph{and}
$\text{final\_conf} \geq \texttt{SIGNAL\_THRESHOLD}$.

\subsection{Walk-forward CV with embargo}

Five expanding-window folds. After each fold's test period, an
\emph{embargo} of $h$ bars is enforced to prevent leakage from
overlapping triple-barrier label horizons.

\section{Risk pipeline}

\subsection{Eight-gate \texttt{approve\_trade}}

\begin{enumerate}
  \item \textbf{Spread filter} (Glosten-Milgrom): reject if
        $\text{spread}_{\text{bps}} > \texttt{SPREAD\_THRESHOLD} \times
        \text{median}_{\text{bps}}$.
  \item \textbf{Daily-loss breaker}: reject all trades for the rest of
        the day if $\text{pnl}_{\text{day}} < -\texttt{MAX\_DAILY\_LOSS\_PCT}$.
  \item \textbf{Drawdown breaker}: full stop if
        $\text{equity} < \text{peak} \cdot (1 - \texttt{MAX\_DRAWDOWN\_PCT})$.
  \item \textbf{Position concentration}: \texttt{MAX\_POSITIONS},
        \texttt{MAX\_POSITION\_PCT} caps.
  \item \textbf{Stoploss guard}: $N$ stop-losses on the same side
        within $T$ hours $\Rightarrow$ pause that side.
  \item \textbf{Long auto-pause}: realised Kelly $f^* < 0$ on the long
        history $\Rightarrow$ block longs unless
        $\text{conf} \geq \texttt{LONG\_MIN\_CONFIDENCE\_WHEN\_BAD\_EDGE}$.
  \item \textbf{Short permission}:
        \texttt{short\_enabled\_flag} \& \texttt{ALLOW\_SHORTS}.
  \item \textbf{Position sizing}: must produce $\geq 1$ lot.
\end{enumerate}

\subsection{Position sizing}

\[
\text{lots}_{\text{Kelly}} = \frac{V \cdot k \cdot c \cdot r}{L \cdot \sigma_{\text{stop}}},\qquad
\text{lots}_{\text{vol}} = \frac{V \cdot v_{\text{target}}}{L \cdot v_{\text{daily}}}
\]
\[
\text{lots}_{\text{final}} = \min\!\bigl(\text{lots}_{\text{Kelly}},\ \text{lots}_{\text{vol}},\ \tfrac{V \cdot p_{\max}}{L}\bigr)
\]

where $V$ is portfolio value, $k = \texttt{KELLY\_FRACTION}$,
$c$ is signal confidence, $r$ is regime scale,
$L$ is lot value (price $\times$ lot size, or initial margin for
futures), $\sigma_{\text{stop}}$ is the stop-loss distance fraction,
$v_{\text{target}} = \texttt{VOL\_TARGET\_DAILY\_PCT}$, $v_{\text{daily}}$
is the realised daily volatility, and $p_{\max} = \texttt{MAX\_POSITION\_PCT}$.

For shorts, $\text{lots}_{\text{final}}$ is multiplied by
\texttt{SHORT\_POSITION\_SCALE} and capped by
$\texttt{SHORT\_CARRY\_FREE\_THRESHOLD\_RUB} / L$.

\subsection{Kyle-Obizhaeva impact}

For large orders ($\text{participation\_rate} > 5\%$), expected slippage
is estimated as
\[
\Delta p = \lambda \cdot \sqrt{q / V_{\text{daily}}}
\]
and \texttt{ExecutionScheduler} splits the order via TWAP slices following
Almgren-Chriss optimal-execution.

The naive approach (polling for \texttt{updater.running}) is insufficient:
the python-telegram-bot updater catches \texttt{NetworkError} internally
and keeps \texttt{running=True} during multi-hour outages. Pinging the
real API via \texttt{get\_me()} is the only reliable liveness signal.
